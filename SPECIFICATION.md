# 栽培技術習得支援ARアプリ 仕様書 v1.0

## 0. このドキュメントの使い方

本仕様書は、Claude Codeを用いてWebARアプリケーションを開発するための要件定義書である。Claude Codeに本ドキュメント全体を読み込ませた上で、開発を依頼することを想定している。

開発開始時は、Claude Codeに対して次のように指示する：「このSPECIFICATION.mdに従ってWebARアプリを開発してください。まず実装方針を提案してから着手してください。」

---

## 1. プロジェクト概要

### 1.1 プロジェクト名（仮）
SeedAR（仮称）

### 1.2 目的
中学校技術科「生物育成の技術」の授業において、生徒の播種実践を支援するWebARアプリを開発する。生徒が選択したプランターサイズと作物に応じて、適切な播種位置がAR表示され、生徒はそれを参考に実際の播種作業を行う。

### 1.3 設計思想
従来の経験的な栽培技術と、先端的な情報技術を橋渡しする「中間地点の体験教材」として機能する。生徒は栽培実践を主軸としながら、情報技術が支援的に関わる体験を通して、両者の融合の在り方を学ぶ。

### 1.4 対象ユーザー
中学校第1学年の生徒（プログラミング学習未経験を前提）

### 1.5 利用シーン
教室内のプランター栽培の現場で、Chromebookのカメラを使って利用する。

## 2. 技術要件

### 2.1 プラットフォーム
WebARアプリケーション（インストール不要、ブラウザで動作）

### 2.2 対象端末・ブラウザ
- 端末：Chromebook
- ブラウザ：Google Chrome（最新安定版）

### 2.3 必須技術スタック

| 領域 | 技術 | 理由 |
|------|------|------|
| AR基盤 | A-Frame + AR.js | マーカーベースARに対応、Chromebookで安定動作 |
| 3D表示 | A-Frame標準コンポーネント | A-Frameに統合済み、追加導入不要 |
| フロントエンド | HTML5 + CSS3 + JavaScript（vanilla） | フレームワーク不要、軽量 |
| 3Dモデル形式 | glTF/GLB | Web標準の3Dフォーマット |
| ホスティング | GitHub Pages | 無料、HTTPS対応（WebAR動作に必須） |

### 2.4 ARの実現方式
**マーカーベースAR**を採用する。プランターの所定の位置に印刷したマーカー（QRコード状のARマーカー）を設置し、それを基準にAR表示を行う。

採用理由：Chromebookは高精度な平面検出センサーを搭載しないため、マーカーベースの方が安定した表示を実現できる。また、生徒が「マーカーを置く」という作業を通して、AR技術の仕組みを直感的に理解できる教育的効果もある。

### 2.5 開発環境（先生のPC側）
- OS：Windows 11
- 必要ツール：Node.js（最新LTS）、Git、VSCode、Claude Code
- ブラウザ：Google Chrome（テスト用）

## 3. 機能要件

### 3.1 機能一覧（v1.0で実装する範囲）

エントリーモデル（2026年7月までの実装）では、**機能①のみを実装**する。機能②③は将来的な拡張として位置づけ、本仕様書では設計の方向性のみ記載する。

### 3.2 機能① 種まき支援機能（必須実装）

#### 3.2.1 機能の流れ
1. アプリ起動 → タイトル画面
2. 「はじめる」ボタンタップ → プランター選択画面
3. プランターサイズを選択（2種類から）
4. 播種方法を選択（すじまき／点まき）
5. AR起動（カメラ起動）
6. プランターに設置したマーカーをカメラに認識させる
7. プランター上に播種位置のARガイドが表示される
8. 生徒は表示された位置に従って播種する

#### 3.2.2 プランターの仕様
本アプリは、以下の2種類の標準的な家庭用プランターに対応する。

| プランターID | 寸法（内寸） | 想定容量 | 一般的な呼称 |
|-------------|-------------|---------|------------|
| size_S | 縦14cm × 横38cm × 深さ18cm | 約10L | ミニプランター |
| size_L | 縦20cm × 横60cm × 深さ20cm | 約25L | 標準プランター |

#### 3.2.3 ラディッシュの播種パターン算出ロジック

**すじまき**の場合：
- 条数：2条固定
- 条間：5cm
- 株間：1cm
- 長辺方向の余白：1cm（両端含めず内側に株を配置）
- 短辺方向の余白：プランターサイズに応じて自動調整（条間5cmを優先）
- 算出式：
  - 1条あたりの株数 = (プランター長辺長 - 余白2cm) ÷ 株間1cm
  - 短辺余白 = (プランター短辺長 - 条間5cm) ÷ 2
- プランターの長辺方向に沿って条を配置
- 株は長辺方向の余白1cmの内側から、株間1cmで配置（両端を含まない内側配置）

**点まき**の場合：
- 株間（穴中心間の距離）：5cm × 5cm（格子状配置）
- 1穴あたり3粒を、3点が小さな三角形（一辺約1cm）を描くように点まきする
- 短辺方向の穴数：size_Sは2点、size_Lは3点
- 長辺方向の余白：2cm（size_S・size_L共通）
- 短辺方向の余白：プランターサイズに応じて自動調整（穴間隔5cmを優先）
- 算出式：
  - 長辺方向の穴数 = floor((プランター長辺長 - 余白4cm) ÷ 5cm) + 1
  - 短辺方向の穴数 = size_Sは2、size_Lは3（固定）
  - 短辺余白 = (プランター短辺長 - 5cm × (穴数-1)) ÷ 2
- 穴の配置開始位置：長辺方向は端から2cm地点、短辺方向は短辺余白の地点

#### 3.2.4 各プランターサイズでの算出結果（確定値）

| プランター | すじまき | 点まき |
|-----------|---------|-------|
| size_S（38×14cm） | 2条 × 36株 = 72株 | 2列 × 7行 = 14穴（42粒） |
| size_L（60×20cm） | 2条 × 58株 = 116株 | 3列 × 12行 = 36穴（108粒） |

**size_Sの詳細**
- すじまき：条間5cm、短辺余白4.5cm × 2、長辺余白1cm × 2、株間1cmで内側配置（両端含まず）
- 点まき：穴間隔5cm、短辺余白4.5cm × 2（2点配置）、長辺余白2cm × 2（穴位置：2,7,12,17,22,27,32cm）

**size_Lの詳細**
- すじまき：条間5cm、短辺余白7.5cm × 2、長辺余白1cm × 2、株間1cmで内側配置（両端含まず）
- 点まき：穴間隔5cm、短辺余白5cm × 2（3点配置）、長辺余白2cm × 2（穴位置：2,7,12,17,22,27,32,37,42,47,52,57cm）

#### 3.2.5 AR表示仕様
- マーカーを基準にプランター上面の3D座標系を構築
- 算出された播種位置に小さな円柱（直径8mm、高さ5mm）の3Dマーカーを配置
- 色：すじまきは緑色、点まきは青色（生徒が選択した方法を視覚的に区別）
- 透過度：50%（実物のプランターと土が透けて見える程度）

#### 3.2.5-bis マーカー設置方法（v1.1 で改訂）

当初は「マーカーをプランター上面（土の表面）に貼付」する想定だったが、実装過程で次の問題が判明したため設置方法を変更した。

**v1.0 案の問題点：**
1. 5cm × 5cm のマーカーを土の上に置くと、種まき位置と物理的に干渉する
2. 家庭用プランターは側面が台形＋上端にリップ（巻き込み）があり、リップの陰でマーカーがカメラから見えなくなる
3. マーカーが土・水で汚れる

**v1.1 採用案：マーカーは机の上、プランター手前に置く**

- マーカー中心はプランター前縁（手前の長辺）から **10cm 前方** の位置
- 専用の **配置シート**（`assets/markers/print.html` で A4 印刷）を使い、プランター配置線とマーカーが固定位置に印刷されているので、生徒が定規で測る必要なし
- シートは切らずに机にそのまま敷き、プランターをシートの上半分に置く運用

**コード上の対応：**
- `js/ar-renderer.js` に `MARKER_FRONT_OFFSET_CM`（既定値 10）を導入
- z 座標計算式が `ar_z = -(planter_y + MARKER_FRONT_OFFSET_CM) / 100` に変更
- 実機で前後ズレを観察した際、教師がこの値を 8〜12 の範囲で微調整できる

**残存する誤差（許容範囲）：**
- マーカーが机の上、土の表面はプランター壁の高さの分だけ上にあるため、AR の高さ方向にプランター壁高さ分のオフセットがある。**真上から見る分にはほぼ知覚されない**ため、種まき作業の精度には影響しない。

### 3.3 機能②③（将来実装、v1.0では実装しない）

機能②：間引きの目安サイズの3Dモデル表示
機能③：収穫の目安サイズの3Dモデル表示

これらは2026年8月以降の実装計画として位置づける。v1.0の設計時には、データ構造に拡張余地を持たせておく（4.1参照）。

## 4. データ構造

### 4.1 作物データ（crops.json）

将来の作物追加に対応するため、JSONファイルで作物情報を管理する。

```json
{
  "crops": [
    {
      "crop_id": "radish",
      "crop_name_ja": "ラディッシュ",
      "sowing_methods": ["row", "point"],
      "row_sowing": {
        "row_count": 2,
        "row_spacing_cm": 5,
        "plant_spacing_cm": 1,
        "long_edge_margin_cm": 1,
        "placement_rule": "internal"
      },
      "point_sowing": {
        "spacing_cm": 5,
        "long_edge_margin_cm": 2,
        "seeds_per_hole": 3,
        "seed_arrangement": "triangle",
        "triangle_side_cm": 1,
        "short_side_holes": {
          "size_S": 2,
          "size_L": 3
        }
      },
      "thinning_size_cm": null,
      "harvest_size_cm": null,
      "thinning_model_path": null,
      "harvest_model_path": null
    }
  ]
}
```

**配置ルールの補足**
- `placement_rule: "internal"` は「両端を含まない内側配置」を意味する。例：長辺38cm、長辺余白1cmの場合、株は2cm, 3cm, ..., 37cmの位置に36株配置される。
- 点まきの`short_side_holes`はプランターIDごとに穴数を指定する。短辺余白はこの値から自動算出される。

※thinning関連、harvest関連のフィールドはv1.0では未使用（null）だが、将来の拡張に備えて構造を確保しておく。

## 5. UI設計

### 5.1 画面遷移

```
[1] タイトル画面
   ↓ 「はじめる」ボタン
[2] プランター選択画面
   ↓ プランター選択
```

### 5.2 各画面の詳細

#### 5.2.1 タイトル画面
- 中央に大きく「SeedAR」（または確定したアプリ名）
- 副題「種まき支援アプリ」
- 「はじめる」ボタン（大きく、タップしやすい）
- 背景色：薄緑（#E8F5E9）

#### 5.2.2 プランター選択画面
- 上部に「プランターのサイズをえらぼう」と表示
- 2つのプランター画像をボタンとして配置
- 各ボタンには「ミニ（38cm×14cm）」「標準（60cm×20cm）」と明記
- 下部に「もどる」ボタン

#### 5.2.3 播種方法選択画面
- 上部に「種まきの方法をえらぼう」と表示
- 2つの方法をイラスト付きボタンで提示
- 「すじまき：一直線にまく」「点まき：等間隔にまく」と説明文
- 下部に「もどる」ボタン

#### 5.2.4 AR表示画面
- 全画面でカメラ映像表示
- 上部に「マーカーをうつしてください」（マーカー認識前）／「種をまきましょう」（認識後）
- 右下に「もどる」ボタン
- 中央下に小さく現在の設定情報（プランター・播種方法）

### 5.3 デザイン原則
- 中学校1年生が直感的に操作できるシンプルなUI
- 文字サイズは18px以上（タイトルは32px以上）
- ボタンの最小サイズは縦60px × 横200px
- 色彩は栽培をイメージする緑系を基調
- 漢字には適宜ふりがなを付ける

## 8. 制約事項

### 8.1 開発上の制約
- 個人開発のため、有料サービスへの依存は最小限とする
- 学校現場での無償利用を前提とする
- ホスティングはGitHub Pagesを利用（無料）

### 8.2 倫理・安全上の配慮
- 生徒の個人情報は取得しない
- カメラ映像はサーバーに送信しない（ローカル処理のみ）
- 利用にあたって個人を特定できる情報の入力を求めない

### 8.3 技術上の制約
- Chromebookの性能上、複雑な3DモデルやアニメーションはARの動作を不安定にする可能性がある。3Dマーカーはシンプルな形状（円柱・球など）に留める。
- マーカー認識には十分な照明が必要。教室環境を想定し、暗所での動作は保証しない。

## 10. 用語集

- **AR（Augmented Reality）**：拡張現実。実物の映像にデジタル情報を重畳表示する技術。
- **マーカーベースAR**：印刷したマーカー画像を基準にAR表示を行う方式。
- **A-Frame**：WebARを実現するためのオープンソースフレームワーク。
- **glTF/GLB**：Web向けの標準的な3Dモデルフォーマット。
- **すじまき**：種を一直線に等間隔でまく播種方法。
- **点まき**：一定間隔で複数粒ずつ穴に種をまく方法。

## 12. 開発開始のための初回プロンプト案

Claude Codeを起動した後、以下のようなプロンプトで開発を始める。

```
このSPECIFICATION.mdの仕様に従って、SeedARというWebARアプリを開発してください。

まずは以下のステップで進めてください。

1. 仕様書全体を読み、不明点や提案があれば質問してください
2. プロジェクトのフォルダ構造を作成してください
3. data/crops.jsonとdata/planters.jsonを作成してください
4. js/calculator.jsで播種位置計算ロジックを実装してください
5. AR以外の3画面（index.html、planter.html、method.html）のUIを実装してください
6. 計算ロジックの単体テストを作成し、動作確認してください

各ステップが完了したら、私に確認を求めてから次に進んでください。
私はプログラミングビギナーですので、専門用語は分かりやすく解説してください。
```
