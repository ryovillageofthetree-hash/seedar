Technology Roadmap

種まき支援AR・公開のしくみ図解

自分のPCで作ったファイルを、どうやって「世界」へ届けるのか？

使う道具とそれぞれの役割

PowerShell

命令を出す「窓口」
黒い画面でコマンドを入力する

Git

履歴を管理する「エンジン」
PC内でファイルを束ねたり送ったりする

GitHub

ネット上の「巨大な倉庫」
世界中のプログラムが集まる場所
公開までの5ステップ：実演とたとえ
A
たとえ話
宅配便の伝票に「自分の名前」を書く
実際の作業
Gitの設定登録
これから行う作業を「誰がやったか」記録するために名前を登録します。
git config user.name "YourName" PowerShell
B
たとえ話
バラバラの原稿を「ホチキス」で一冊に綴じる
実際の作業
コミット作業
編集中のファイルを1つのセットとして保存。後でこの時点に戻せるようになります。
git commit -m "ver1.0" PowerShell
C
たとえ話
ナビに「目的地の住所」を入力する
実際の作業
リモート倉庫の登録
自分のPCのGitに、送信先となるGitHub上の倉庫の住所を教えます。
git remote add origin https://... PowerShell
D
たとえ話
トラックで荷物を「倉庫」まで運ぶ
実際の作業
プッシュ送信
重要！
PowerShellの命令で、Gitがインターネットを通じてファイルを送信します。
git push origin main PowerShell
E
最後にブラウザで設定

GitHub Pagesを有効化

倉庫に届いたファイルを「公開」に設定すると、世界中で見られるURLが発行されます。

一連の流れを「たとえ」でおさらい！

Identity

Step E
お店に並ぶ
Pages
