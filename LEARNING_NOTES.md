# 種まき支援AR が世界に公開されるまで

## ＿＿＿＿＿＿＿＿ さんむけ・ホームページ公開のしくみ

私たちが Chromebook で使っている「種まき支援AR」は、はじめからインターネットにあったわけではありません。
先生のパソコンで作ったアプリを、**5 つのステップ** で世界中に公開しました。

公開先：<https://ryovillageofthetree-hash.github.io/seedar/>

このプリントでは、そのしくみを **たとえ** を使って説明します。

---

## 1. 5 人の登場人物

| 登場人物 | しごと | たとえると |
|---|---|---|
| **パソコン** | アプリのファイルが入っている場所 | 編集中の原稿のかたまり |
| **PowerShell**（黒い画面） | パソコンに命令を出す窓口 | お店の **注文カウンター** |
| **Git** | ファイルの履歴を管理するソフト | 「いつでも前のバージョンに戻せる **タイムマシン**」 |
| **GitHub** | インターネット上のファイル倉庫 | コード専用の **クラウド倉庫** |
| **GitHub Pages** | 倉庫のファイルをホームページにする機能 | クラウドに届いた本を **書店に並べる工場** |

---

## 2. 5 ステップでやったこと

### Step A：Git に「差出人」を教える

```
git config user.name "ryovillageofthetree-hash"
git config user.email "..."
```

→ あとで「だれがこのファイルを作ったか」を残せるようになる。  
**たとえ：宅配の伝票に **差出人の名前** を書く作業。**

---

### Step B：いまのファイル全部を「第 1 版」にまとめる（コミット）

```
git commit -m "initial commit: SeedAR v1.0"
```

→ パソコンの中にあるたくさんのファイルを、ひとまとめの「セット」にする。  
**たとえ：書きかけの原稿を全部 **ホチキスで束ねて** 「2026年 1 月号・第 1 版」とラベルを貼る。**

---

### Step C：GitHub の倉庫を「送り先」として登録

```
git branch -M main
git remote add origin https://github.com/.../seedar.git
```

→ Git に「これから送るときは、ここに送ってね」と教える。  
**たとえ：宅配の集荷依頼で **届け先の住所** を伝える作業。**

---

### Step D：いよいよインターネットに送信（push）

```
git push -u origin main
```

→ PowerShell が Git に「送って」と命令、Git がインターネット越しに GitHub に荷物を届ける。途中でブラウザが開いて「これはあなたですか？」と本人確認も自動で済ませてくれる。  
**たとえ：宅配ボックスにポンと荷物を入れる。**

---

### Step E：GitHub Pages を有効にして「公開」する（クラウドだけで完結）

GitHub のサイトで：
**Settings → Pages → main / root → Save**

→ GitHub に「このファイル群を、世界中のだれもが見られる URL にしてね」とお願いする。  
**たとえ：倉庫に届いた本を、**全国の書店に並べる手続き**。**

## 4. 大事なポイント

- 自分のパソコンで作ったプログラムは、**ただ作っただけでは世界の誰も見られません**。
- PowerShell → Git → GitHub → GitHub Pages の **4 つを順番につないだ瞬間** に、それが世界中の Chromebook で動くようになります。
- たとえとしては、原稿を書く → 本にまとめる → 印刷所に送る → 書店に並べる、という流れに近いものです。
- このしくみが **無料** で使えるのが、GitHub と GitHub Pages のすごいところです。世界中の多くのウェブアプリやライブラリは、この同じ仕組みを使って公開されています。
